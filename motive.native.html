Motive Native Module

>>>update()

motive.native.flush_camera_queues()¶

In the event that you are tracking a very high number of 2D and/or 3D markers (hundreds of 3D markers), and you find that the data you’re getting out has sufficient latency you can call flush_camera_queues() to catch up before calling update(). Ideally, after calling flush_camera_queues() you’ll want to not call it again until after update() returns a frame again.
